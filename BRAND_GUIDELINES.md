## Typography

### Brand Font Stack

```
-apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 'Oxygen',
'Ubuntu', 'Cantarell', 'Fira Sans', 'Droid Sans', 'Helvetica Neue', sans-serif
```

**Rationale**: System fonts ensure consistency across devices and improve performance.

### Type Hierarchy

| Level | Size | Weight | Usage |
|-------|------|--------|-------|
| **H1** | 2.5-3rem | 800 (Extra Bold) | Page titles, hero sections |
| **H2** | 2rem | 700 (Bold) | Section headers |
| **H3** | 1.4-1.5rem | 600 (Semibold) | Subsection headers |
| **H4** | 1.2rem | 600 (Semibold) | Card titles, highlights |
| **Body** | 1.05rem | 400 (Normal) | Main text content |
| **Caption** | 0.85-0.9rem | 500 (Medium) | Labels, metadata |

### Best Practices

- **Headings**: Use weight 700 or higher, maintain tight line-height (1.2-1.3)
- **Body Text**: Weight 400, line-height 1.8 for readability
- **Line Length**: Keep between 50-75 characters for optimal reading
- **Contrast**: Ensure sufficient contrast for accessibility

## Voice & Tone

### Brand Voice

VegaKash.AI communicates with:

- **Clarity**: Simple, jargon-free language
- **Confidence**: Assured and knowledgeable without being pushy
- **Friendliness**: Approachable and supportive
- **Expertise**: Demonstrating deep financial knowledge

### Tone Variations

| Context | Tone | Example |
|---------|------|---------|
| Educational | Informative, patient | "Let's walk through your income sources..." |
| Marketing | Engaging, empowering | "Take control of your finances today" |
| Errors | Helpful, reassuring | "Oops! Let's fix that together" |
| Success | Celebratory, positive | "Great! Your budget is ready" |

### Writing Guidelines

‚úÖ **DO:**
- Use second person ("you") to address users
- Keep sentences short and clear
- Use active voice
- Explain features in terms of user benefits
- Use relatable examples

‚ùå **DON'T:**
- Use corporate jargon
- Make assumptions about user knowledge
- Use passive voice
- Be condescending or overly casual
- Ignore accessibility in copy

## Design Principles

### 1. **Trust Through Transparency**
- Clearly communicate how AI works
- Show calculations transparently
- Explain recommendations with reasoning
- Never hide fees or limitations

### 2. **Simplicity Over Complexity**
- One clear action per screen when possible
- Progressive disclosure of advanced features
- Obvious visual hierarchy
- Whitespace as design element

### 3. **Human-Centered Technology**
- AI assists human decision-making, not replaces it
- Clear error messages with solutions
- Helpful guidance, not strict rules
- Celebrate user achievements

### 4. **Consistency Builds Trust**
- Consistent patterns across all pages
- Predictable interactions
- Unified visual language
- Reliable performance

### 5. **Accessibility is Mandatory**
- All users should have equal access
- Keyboard navigation everywhere
- Clear focus indicators
- Sufficient color contrast

## Photography & Imagery

### Style

- **Professional yet approachable**: Not sterile, but polished
- **Diverse representation**: Show real people from different backgrounds
- **Action-oriented**: Show people using products, not just posing
- **Authentic**: Real situations, not overly staged

### Image Types

1. **Hero Images**: Large, inspiring, showing financial success/peace
2. **Icons**: Clean, geometric, consistent with brand aesthetic
3. **Illustrations**: Minimal, geometric style with brand colors
4. **Screenshots**: Clean, well-lit, showing actual product

### Image Guidelines

- **Resolution**: Minimum 2x resolution for retina displays
- **File Size**: Optimize for web (use modern formats like WebP)
- **Aspect Ratios**: 
  - Hero: 16:9 or 4:3
  - Cards: 4:3 or 1:1
  - Social: 1.2:1 (Facebook), 16:9 (Twitter)

## Interactive Elements

### Buttons

**Primary Button**
- Background: Purple gradient (#667eea ‚Üí #764ba2)
- Text: White, bold (600-700 weight)
- Padding: 0.875rem 2rem
- Border Radius: 8px
- Shadow: Subtle shadow on default, lifted on hover
- Min Height: 44px (touch target)

**Secondary Button**
- Background: White
- Border: 2px solid #e8ecf3
- Text: Purple (#667eea), bold
- Same sizing as primary

**Hover State**: Lift effect (translateY -2px), enhanced shadow

### Forms

**Input Fields**
- Border: 2px solid #e8ecf3 (default)
- Border: 2px solid #667eea (focused)
- Padding: 0.75rem 1rem
- Font: Inherit from body
- Min Height: 44px

**Labels**
- Above input (not placeholder)
- Color: Dark gray (#475569)
- Weight: 500 (medium)
- Size: 0.9rem

### Micro-interactions

- **Hover**: Smooth color/shadow transitions
- **Focus**: 2px outline in brand purple
- **Loading**: Subtle spinner animation
- **Success**: Brief checkmark animation
- **Error**: Shake animation with error color

## Do's & Don'ts Summary

### Design Do's ‚úÖ

1. Use brand colors consistently
2. Maintain 4.5:1 contrast ratio
3. Keep layouts clean and spacious
4. Use system fonts for performance
5. Test on mobile and desktop
6. Follow established patterns
7. Use brand assets officially
8. Include alt text for images
9. Make interactive elements obvious
10. Test animations on various devices

### Design Don'ts ‚ùå

1. Don't mix multiple gradients
2. Don't use non-approved colors
3. Don't create duplicate components
4. Don't ignore accessibility
5. Don't use heavy animations
6. Don't hardcode values
7. Don't stretch/distort logo
8. Don't use Comic Sans (always!)
9. Don't override established patterns
10. Don't assume mobile users
